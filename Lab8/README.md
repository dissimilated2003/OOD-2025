# паттерн Состояние

#### поведенческий паттерн проектирования, где класс контекст хранит в себе состояния, а состояния являются отдельными классами, и контекст внутри себя их переключает
паттерн призван избавить от чудовищного нагромождения проверок на текущие состояния, призван сделать код чище, а также инкапсулировать в себе стратегии переключения состояний (никто не может взять залезть внутрь и переключить на нужное состояние)
