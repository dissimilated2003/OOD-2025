# паттерн Компоновщик

#### структурный паттерн проектирования, позволяющий представить объекты в древовидной структуре для отображения иерархии "часть - целое"
**Component** - это абстракция над листом и узлом, **Leaf** - листовой объект, **Composite** - группа, включающая в себя либо группы, либо листовые объекты. главное преимущество паттерна в том, что клиентский код не знает, с чем он работает, Leaf и Composite реализуют интерфейс, который использует клиент, это избавляет от всяких парашных проверок  
<br> крч как сильноветвящееся дерево из алгоритмов, только упакованное в SOLID
