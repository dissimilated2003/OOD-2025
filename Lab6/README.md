# паттерн Адаптер

#### структурный паттерн проектирования, позволяющий работать друг с другом несовместимым интерфейсам
есть 2 штуки, адаптер объекта и адаптер класса. адаптер объекта более гибкий, т.к. реализует интерфейс, адаптер класса менее гибкий, т.к. реализует конкретный класс, затаскивая его методы в приватку, но хорошо работает с его наследниками
